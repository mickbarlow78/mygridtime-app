## 1. Critical UX Issues

1. **Admin header overflows horizontally at 375px mobile.** Body `scrollWidth` = 588 at viewport 375 on `/admin`. Flex row (`flex items-center` with `flex-wrap: nowrap`) of logo + OrgSelector + Events/Settings/+ New org + UserMenu badge cannot compress. Admin is effectively unusable on phone — forces horizontal scroll on every screen. Same overflow visible on `/admin/events/[id]` (588 > 375). `/my` is clean (375 = 375). _Resolved in MGT-094 (2026-04-22): `HEADER_INNER` now wraps at `<640px`, right-group gets `min-w-0 flex-wrap`, `document.body.scrollWidth === window.innerWidth === 375` verified on `/admin`._
2. **Tap targets fail WCAG 2.5.5.** UserMenu badge button: 24px tall. "Events"/"Settings" nav links: 16px tall. Sign-out in popover: tiny underlined text link. Min target = 44×44. None of the admin chrome is operable one-handed on a phone. _Header chrome resolved in MGT-094 (2026-04-22): `OrgSelector`, `HEADER_NAV_LINK`, and the `UserMenu` badge now report 40px height on mobile with desktop sizing preserved via `sm:` overrides. In-popover sign-out + org-switch rows resolved in MGT-095 (2026-04-22) — all three popover interactive items (org rows, empty-org "Create an organisation" link, sign-out) now 40px on mobile with byte-identical desktop presentation via `sm:` overrides; sign-out promoted to a real padded button on mobile per Suggested Improvement #10._
3. **MemberManager lets a user remove themselves.** The self-row renders a "Remove" button with no guard. If the user is the sole owner, nothing stops them bricking the org (needs server-side verification, but the client offers no warning). _Resolved in MGT-098 (2026-04-23): `MemberManager` now accepts a `currentUserId` prop threaded from [src/app/admin/orgs/settings/page.tsx](src/app/admin/orgs/settings/page.tsx); the per-row `Remove` button is `disabled` + `aria-disabled` with title "You cannot remove yourself from this organisation." when `member.user_id === currentUserId`; `removeMember` in [src/app/admin/orgs/actions.ts](src/app/admin/orgs/actions.ts) enforces the same rule server-side as defence in depth, firing before the existing last-owner guard._
4. **Two separate Save buttons on event editor (details + timetable) with no unified commit.** Classic "did I save everything?" pitfall — user fills metadata, drags timetable rows, hits one button, loses the other half. No visual indication of unsaved state per section. _Resolved in MGT-096 (2026-04-23): single primary `Save changes` button in `EventActionsBar`; `reviewMode` extended with `'unified'` so one Review modal pass lists metadata cards then timetable cards under a `Review changes` title; `runUnifiedSave()` runs `performMetaSave` → `performTimetableSave` sequentially; one `Changes saved.` banner; DEC-002 notify semantics preserved. UI+orchestration only across 3 files; no server-action change._
5. **Timetable entry rows have no column headers or labels.** Each row is 5 adjacent inputs (Title*, start time, end time, Category, Notes) with placeholder-only labels. Second row onwards, user relies on position memory. Not accessible.
6. **`/my` cannot switch orgs without opening UserMenu popover.** No OrgSelector in the `/my` header, no inline org context on the "My Timetables" heading. Discoverability gap for multi-org consumers. _Resolved in MGT-097 (2026-04-23): `OrgSelector` now renders in the `/my` header left-group under the same `userOrgs.length > 1 && activeOrg` gate used by `/admin`, with the same `max-w-[160px] truncate` + MGT-094 mobile tap-target sizing; switch behaviour reuses the existing `switchOrg()` action + `router.refresh()` — no logic changes._

## 2. Friction / Inefficiencies

7. **Event-editor action bar stacks five equal-weight primary buttons:** View public · Unpublish · Archive · Duplicate · Save as Template. Destructive (Archive, Unpublish) sits alongside construction (Duplicate, Save as Template). No hierarchy, no grouping, no colour differentiation.
8. **Character counters always rendered even on empty inputs.** `0/120`, `0/254`, `0/1000`, `0/80` clutter every form before the user types anything.
9. **`7/7` counter on the Primary colour hex input** — fixed-length field, counter adds zero value.
10. **Redundant create-event CTAs in empty state.** Org B empty dashboard shows both "+ Create event" top-right AND "Create your first event →" in the list — same destination, double CTA.
11. **"From template" mode invisible on `/admin/events/new` when the org has no templates.** A brand-new owner never learns templates exist. The mode tab should show disabled-with-hint, not be omitted.
12. **`/admin/orgs/settings` breadcrumb says "Events / Organisation settings"** — settings isn't an event. Parent crumb is wrong.
13. **Three collapsible panels at event editor bottom use three different styles.** "Version History (1)" (chevron icon, title case), "Audit log 3 entries ▼" (unicode arrow, sentence case), "Notification history ▼" (no count at all). Same pattern, three renderings.
14. **Extraction log panel on org settings has no entry count** — inconsistent with Audit log ("15 entries"), Notification log, and Version History panels which do show counts.
15. **Display name falls back to email prefix (`mickbarlow`).** With no first-name fallback logic visible, every user without a display name sees a lowercase email-local-part. Feels impersonal at the top of the popover.

## 3. Visual / UI Issues

16. **"Audit log15 entries ▼"** — no space between label and count. Same concat bug on `/admin/orgs/settings` and event editor. Visual AND accessibility (screen reader reads "Audit log fifteen entries").
17. **Inconsistent button capitalisation.** Title case: "Save as Template", "Version History", "+ New org". Sentence case: "Save details", "Save timetable", "Save branding", "Save invite", "Create event". Pick one.
18. **Unicode glyph icon buttons lack hover tooltips.** ⧉ has `title="Duplicate row"`, but ✕ has `title=""` (empty) and ⠿ has no `title` at all. Aria-labels cover screen readers; desktop mouse users have no discovery path.
19. **UserMenu popover heading "ORGANISATIONS" is uppercase small-caps** while Role row heading ("Role") is sentence case. Stylistic isolation.
20. **Public org table has 5 columns (Event · Venue · Start · End · View) dominated by `—` placeholders** for single-day, venue-less rows. Over-structured for the data volume.
21. **Public event page renders day label "Fri 1 May" twice** for single-day events — once as a section header, once as the solo day tab.
22. **Version badge visible on public routes leaks commit hash `20cc266`** to consumers. Internal artifact polluting the public surface.
23. **Members list is a flex row, not a `<table>`** — no `<th>`, no semantic column structure. Role `<select>` + "Remove" button flow inline with date text.
24. **Two date representations across the app** — "Joined 17/04/2026" (DD/MM/YYYY numeric) in Members, "Fri 1 May" (short weekday + month) in event listings. Both valid, both used, inconsistent.

## 4. Terminology Problems

25. **"Event" vs "Timetable" used interchangeably for the same entity.** `/admin` = "Events" / "Create event" / "Manage your race event timetables". `/my` = "My Timetables". Public org = "Published timetables from this organisation". Public event page = just the title. Pick one noun.
26. **"Member" is overloaded.** MemberManager = *org member*. UserMenu subscription row = *free-tier consumer* ("Subscription: Member"). Two different concepts, same word.
27. **Badge vs popover role mismatch feels like two different role states.** Pill: `Staff — MGT-060 Verify Org`. Popover Role row (post-MGT-091): `Staff`. Read together, it looks like "Staff" is different from "Staff — {org}" rather than the same role presented with/without context.
28. **Timezone option `Europe/London (UK)`** — the "(UK)" decoration is redundant given the identifier.
29. **Breadcrumb uses "Events" as root** for both event editor (correct) and Organisation settings (wrong). Needs a neutral root ("Admin") or separate roots per subtree.
30. **Create-event mode labels: "From scratch" / "From PDF / image"** — two different grammatical shapes. If/when "From template" returns, it's a third shape.

## 5. Suggested Improvements (prioritised, ticket-sized)

1. **MGT-NEW · CRITICAL — Admin header mobile layout.** Wrap/collapse to hamburger below ~640px. Fixes items 1, 2. _Resolved in MGT-094 (2026-04-22) via `flex-wrap` + mobile-only tap-target sizing — header-chrome scope; in-popover sign-out remains under improvement #10._
2. **MGT-NEW · HIGH — Unify event save into a single top-of-editor "Save" commit** covering both details and timetable; strip the two existing save buttons. _Resolved in MGT-096 (2026-04-23): `Save changes` in `EventActionsBar`; the two legacy buttons and their per-section success banners removed; Enter-in-details-form now commits the full editor via `handleUnifiedSave`._
3. **MGT-NEW · HIGH — Event/Timetable terminology pass.** Pick one entity noun and sweep `/admin`, `/my`, public surface, email templates.
4. **MGT-NEW · HIGH — Add OrgSelector (or inline active-org chip) to `/my` header.** Mirror the `/admin` switching affordance. _Resolved in MGT-097 (2026-04-23): reuse of the existing `admin/OrgSelector` component in the `/my` layout — one file, one new import, one new conditional element, two className edits; no new component, no action change._
5. **MGT-NEW · HIGH — Self-remove guard on MemberManager.** Disable Remove on own-row; disable on last-owner row; add confirm copy for owner demotion. _Resolved in MGT-098 (2026-04-23): own-row `Remove` hard-disabled in UI + hard-rejected in `removeMember` server action. Last-owner guard retained as fallback (already in place pre-MGT-098). Owner-demotion confirm copy already exists via `updateMemberRole` / `ConfirmDialog` downgrade path (out of MGT-098 scope; not changed)._
6. **MGT-NEW · MEDIUM — Entry row column headers.** Add a visible/labelled header row on the timetable grid; add `aria-label` on every input.
7. **MGT-NEW · MEDIUM — Collapsible-panel unification.** One chevron, one "Label (N entries)" format, one heading style across Version History / Audit log / Notification history / Extraction log.
8. **MGT-NEW · MEDIUM — Button-casing pass.** Sentence-case everywhere; rename "Save as Template" → "Save as template", "Version History" → "Version history", "+ New org" → "+ New organisation".
9. **MGT-NEW · MEDIUM — Event-editor action-bar hierarchy.** Primary ("View public"), secondary (Duplicate, Save as template), destructive group (Unpublish, Archive) with a subtle separator or kebab menu.
10. **MGT-NEW · MEDIUM — Tap-target sweep.** Every header control, every icon button ≥ 40×40px on mobile; sign-out button → real button, not underlined text. _UserMenu popover portion resolved in MGT-095 (2026-04-22). Remaining scope: non-header icon buttons (✕ delete, ⠿ drag, ⧉ duplicate) on timetable rows, event editor action bar, etc. — separate future ticket._
11. **MGT-NEW · LOW — `/admin/events/new` "From template" tab always visible**, disabled with tooltip when empty.

5. **Add entry count to Extraction log and Notification history panel buttons.**
6. **Drop "Events" parent crumb on `/admin/orgs/settings`** or change it to "Admin".
7. **Hide the build-identity badge on `/(public)/*` routes** (or drop the hash portion there). Keeps internal context out of the consumer view. _Resolved in MGT-093 (2026-04-22)._
8. **Collapse duplicate "Fri 1 May" header on public event page** when there's only one day.
9. **Drop `(UK)` from the default timezone option label.**
10. **Replace empty-state double CTA** ("+ Create event" top-right + "Create your first event →" in-list) with one explicit CTA in the empty state.
11. **First-name fallback on UserMenu display name** — use `users.display_name` → `first_name + last_name` → email local-part capitalised, not raw `mickbarlow`.
